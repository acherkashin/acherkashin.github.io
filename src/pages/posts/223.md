[size-limit](https://github.com/ai/size-limit)

[Вот здесь](https://t.me/cherkashindev/220) я писал про [source-map-explorer](https://create-react-app.dev/docs/analyzing-the-bundle-size/) и о том, как с его помощью анализировать размер бандла. Но, чтобы не приходилось анализировать бандл, лучше предотвратить его раздувание.

Это можно сделать на этапе CI с помощью [size-limit](https://github.com/ai/size-limit).


**Как использовать❓**

1. Устанавливаем `size-limit` и плагин `@size-limit/file`, необходимый для анализа web-приложений.

```shell
npm install --save-dev size-limit @size-limit/file
```

2. Добавляем когфиг size-limit в `package.json`

```json
  "size-limit": [
    {
      "limit": "35 kB",
      "path": "dist/app-*.js",
      "gzip": true
    }
  ],
```

3. Добавляем запуск size-limit в скрипт, который вы запускаете на CI

```json
  "scripts": {
    "size": "npm run build && size-limit",
    "test": "vitest && eslint . && npm run size"
  }
```

Если вы пользуетесь гитхабом, то можете использовать [GitHub action](https://github.com/andresz1/size-limit-action), который добавляет в качестве комментария изменение размера бандла.

Также size-limit поддерживает задание [временных лимитов](https://github.com/ai/size-limit/?tab=readme-ov-file#js-application-and-time-based-limit) на исполнение вашего кода и использует для этого “безголовый хром”.
